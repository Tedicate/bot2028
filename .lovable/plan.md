

## 문제 분석

1. **후속 안내의 예시 키워드가 클릭 불가** — "기하", "역학과 에너지", "컴퓨터공학과" 등이 일반 텍스트로만 표시됨
2. **체크박스가 텍스트로 렌더링** — `[ ] 경희대학교 우주과학과`가 마크다운 원문 그대로 노출 (스트리밍 중 `- [ ]` 패턴 매칭 실패 가능성)
3. **첫 페이지 SUGGESTIONS가 학과명만** — "OO대학교 XX학과" 형태 + 과목명 버튼도 필요

## 해결 계획

### 1. 후속 안내에 클릭 가능한 버튼 추가

**시스템 프롬프트 수정 (`course-advisor/index.ts`):**
- 후속 안내 섹션에 특수 마커를 추가: `<!--SUGGEST:기하-->`, `<!--SUGGEST:역학과 에너지-->`, `<!--SUGGEST:컴퓨터공학과-->` 형식
- AI가 답변 마지막에 이 마커를 포함하도록 프롬프트 지시

**프론트엔드 파싱 (`ChatBot.tsx`):**
- `renderAssistantContent`에서 `<!--SUGGEST:XXX-->` 패턴을 감지하여 클릭 가능한 버튼으로 렌더링
- 버튼 클릭 시 해당 텍스트로 `send()` 호출

### 2. 체크박스 렌더링 수정

현재 `checkboxPattern`이 `/^- \[ \] (.+)$/gm`인데, 스트리밍 완료 후에도 매칭이 안 되는 경우가 있음. 원인:
- 스트리밍 중간에 `renderAssistantContent`가 호출되면 불완전한 텍스트에서 패턴 매칭 실패
- 한번 일반 마크다운으로 렌더링되면 이후 업데이트에서도 체크박스로 전환되지 않을 수 있음

**수정:**
- 체크박스 패턴 매칭을 더 유연하게: `\[ \]` 앞의 `- ` 없이도 매칭되도록 (`/^\[[ ]\] (.+)$/gm` 추가)
- 스트리밍 완료 여부(`isLoading`)를 `renderAssistantContent`에 전달하여, 스트리밍 완료 후 re-render 보장

### 3. 첫 페이지 SUGGESTIONS 업그레이드

`SUGGESTIONS` 배열을 두 섹션으로 분리:

**학과 추천 (대학+학과 형태):**
- "서울대 컴퓨터공학부", "연세대 경영학과", "고려대 전기전자공학부", "성균관대 의예과", "한양대 기계공학부" 등 서울 주요대 인기학과

**과목 추천 (진로/융합선택 위주):**
- "미적분II", "기하", "역학과 에너지", "물질과 에너지", "생명과학과 지구시스템", "정보", "인공지능 기초", "데이터 과학"

UI에서 "🎓 인기 학과" / "📚 주요 과목" 라벨로 구분하여 표시.

### 수정 파일
1. **`src/components/ChatBot.tsx`** — SUGGESTIONS 업그레이드, SUGGEST 마커 버튼 렌더링, 체크박스 매칭 개선
2. **`supabase/functions/course-advisor/index.ts`** — 시스템 프롬프트에 `<!--SUGGEST:XX-->` 마커 지시 추가
